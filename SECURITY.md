# Security Policy

## Supported Scope

This public repository contains documentation, examples, public schemas, placeholder API contracts, and GitHub metadata.

Do not report vulnerabilities in public GitHub issues. Use responsible disclosure for:

- Public API authentication or authorization issues.
- Token handling concerns.
- Documentation that could cause insecure integration behavior.
- Security-sensitive schema or example problems.
- Suspected exposure of proprietary, private, or sensitive material.

## Reporting a Vulnerability

Email: security@bcinexus.com

Include:

- Affected file, endpoint, schema, or example.
- Reproduction steps.
- Potential impact.
- Suggested remediation if known.

Please avoid including real user data, private keys, credentials, or protected research data in reports.

## Response Expectations

We aim to acknowledge valid reports within 5 business days. Timing for remediation depends on severity, scope, and whether the issue affects public documentation only or a production service.

## Public Disclosure

Please do not publicly disclose vulnerabilities before coordinated remediation.
