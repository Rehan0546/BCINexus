# Brand Guidelines

## Name

Use `BCINexus` for the developer and cloud ecosystem.

Use `BCILattice` for the local research application.

## Positioning

BCINexus should be described as a developer ecosystem, cloud layer, and public integration surface for BCILattice workflows.

## Do

- Use official names exactly.
- Keep the distinction between BCINexus and BCILattice clear.
- Use assets in documentation, examples, and integration references.

## Do Not

- Imply BCINexus or BCILattice source code is included in this repository.
- Imply partnership, endorsement, certification, or hardware compatibility without written approval.
- Modify logos in ways that make them misleading.
